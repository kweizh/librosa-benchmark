## Approach

The engine uses **subsequence matching** with MFCC-based frame-level features:

1. **Frame-level features**: For each audio file, 20 MFCCs + delta + delta-delta (60 features per frame) are extracted using librosa 0.11.0.
2. **Segment embeddings**: For a given segment, mean and standard deviation are computed across all frames, producing a 120-dimensional vector that is L2-normalized.
3. **Subsequence matching**: Since the query is a short clip (5s) from a longer track (15s), the query's segment length (in frames) is used as a sliding window across each library track. The **maximum** cosine similarity across all window positions is taken as the track's similarity score.
4. **Ranking**: All 6 library tracks are ranked by descending cosine similarity.
