**How the algorithm works:**

1. **Load** — both files at the same `SR=22050` Hz via `librosa.load`.

2. **Spectrogram** — `librosa.stft` (N_FFT=4096, hop=512) → `amplitude_to_db` for a log-magnitude 2-D array.

3. **Constellation map** — `scipy.ndimage.maximum_filter` finds 2-D local maxima in a 15×9 (freq×time) neighbourhood. A global −30 dB threshold removes weak peaks; only the top 5 peaks per time frame are kept. This gives ~3 800 database peaks and ~66 query peaks (the noise reduces query peak count).

4. **Hashing** — each anchor peak is paired with up to 15 later peaks inside a bounded target zone (`DT ∈ [1, 80]` frames, `|DF| ≤ 100` bins). The hash key is `(f_anchor, f_target, Δt)`.

5. **Offset voting** — for every hash shared between the database and query fingerprints, `offset = t_db − t_query` is accumulated in a histogram. A box-car smoothing over ±1 adjacent bin is applied; the peak bin gives the winning time offset.

6. **Output** — offset converted to seconds, clamped to ≥ 0, written to `match.json`. Confidence = fraction of all matched hash pairs that voted for the winning offset.
