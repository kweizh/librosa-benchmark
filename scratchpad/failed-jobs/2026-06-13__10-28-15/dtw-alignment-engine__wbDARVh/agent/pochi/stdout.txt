### What was built

**`/workspace/align.py`** follows these steps:

1. **Load audio** — both WAV files are loaded at 22 050 Hz mono via `librosa.load`.
2. **CENS chroma features** — `librosa.feature.chroma_cens` is called with `hop_length=512` (~23 ms/frame) and a 41-frame smoothing window, producing 12-bin chroma matrices robust to timbre and tempo differences.
3. **DTW** — `librosa.sequence.dtw` with cosine distance (a tiny ε=1e-8 is added to every frame to prevent NaN from all-zero frames) aligns the two chroma sequences.
4. **Path reversal** — the warping path (returned end→start by librosa) is reversed so it runs start→end.
5. **Frame → seconds** — every frame index is multiplied by `hop_length / sample_rate`.
6. **JSON output** — the full path (3 295 entries) is serialised as an array of `{ref_time, perf_time}` objects.
