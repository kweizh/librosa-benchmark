## What was built

**`/workspace/search.py`** — the search engine, runnable with `python3 search.py`.

### Algorithm

**Frame-level STFT magnitude + sliding window**

1. Load the query clip (`query.wav`, 5 s) and compute its L2-normalised STFT magnitude vector (flat `(F × T,)` array).
2. For each library track, slide a window of the same duration across the track in 10 % steps, compute the cosine similarity of each window's STFT vector to the query vector, and keep the **maximum** window similarity as the track's score.
3. Sort all tracks by score descending and emit `results.json`.

The key insight: aggregate statistics (MFCC means, etc.) capture genre/style but discard fine-grained spectral identity. The flat STFT magnitude vector preserves exact time-frequency content, so the window that contains the actual query clip scores ~0.987, while the next-best unrelated track scores only ~0.33 — a clear separation.
