### How it works

**Streaming parameters (contract)**
| Parameter | Value |
|---|---|
| `block_length` | 16 frames |
| `frame_length` | 2048 samples |
| `hop_length` | 512 samples |

**Sample-offset bookkeeping**

`librosa.stream` yields 1-D NumPy arrays of raw audio samples. Each full block spans:
```
samples_per_block = frame_length + (block_length - 1) × hop_length
                  = 2048 + 15 × 512 = 9 728 samples
```
Consecutive blocks start `block_length × hop_length = 8 192` samples apart (standard overlap-add stride). Block `i` therefore occupies:
```
start_sample = i × 8192
end_sample   = start_sample + len(block)   # len(block) ≤ 9728
```
Dividing by the sample rate (read via `librosa.get_samplerate`, never hard-coded) gives the time columns.

**RMS computation**
```python
rms = sqrt(mean(block ** 2))
```
Applied directly to the raw sample array for each block, with `center=False` semantics naturally satisfied because no frame-based STFT analysis is called on the block.
