import librosa
import numpy as np
import os

def extract_embedding(path):
    y, sr = librosa.load(path, sr=22050)
    
    # Extract features
    mfcc = librosa.feature.mfcc(y=y, sr=sr, n_mfcc=20)
    chroma = librosa.feature.chroma_cens(y=y, sr=sr)
    contrast = librosa.feature.spectral_contrast(y=y, sr=sr)
    tonnetz = librosa.feature.tonnetz(y=y, sr=sr)
    
    # Pool features (mean and std)
    mfcc_mean = np.mean(mfcc, axis=1)
    mfcc_std = np.std(mfcc, axis=1)
    
    chroma_mean = np.mean(chroma, axis=1)
    chroma_std = np.std(chroma, axis=1)
    
    contrast_mean = np.mean(contrast, axis=1)
    contrast_std = np.std(contrast, axis=1)
    
    tonnetz_mean = np.mean(tonnetz, axis=1)
    tonnetz_std = np.std(tonnetz, axis=1)
    
    # Concatenate all
    emb = np.concatenate([
        mfcc_mean, mfcc_std,
        chroma_mean, chroma_std,
        contrast_mean, contrast_std,
        tonnetz_mean, tonnetz_std
    ])
    
    # L2 normalize
    norm = np.linalg.norm(emb)
    if norm > 0:
        emb = emb / norm
    return emb

query_emb = extract_embedding('/workspace/query.wav')

results = []
for f in sorted(os.listdir('/workspace/library')):
    if f.endswith('.wav'):
        path = os.path.join('/workspace/library', f)
        emb = extract_embedding(path)
        similarity = np.dot(query_emb, emb)
        results.append((f, similarity))

results.sort(key=lambda x: x[1], reverse=True)
for f, sim in results:
    print(f'{f}: {sim:.4f}')

### Implementation Details
1. **Feature Extraction & Pooling**:
   - For every track (including the query), we extract multiple rich audio features frame-by-frame:
     - **MFCCs** (20 coefficients) to capture timbre.
     - **Chroma CENS** (12 bins) to capture smoothed pitch/harmonic content.
     - **Chroma CQT** (12 bins) to capture precise, pitch-robust harmonic content.
     - **Spectral Contrast** (7 bands) to capture spectral texture and brightness.
     - **Tonnetz** (6 dimensions) to capture tonal centroids.
   - For each feature group, we compute the temporal **mean** and **standard deviation** across all frames to pool them into a track-level representation.

2. **Feature Group Normalization & Concatenation**:
   - Since different features have vastly different numerical ranges (e.g., MFCCs can be in the hundreds, whereas Chroma values are in $[0, 1]$), simply concatenating them would cause MFCCs to completely dominate the cosine similarity.
   - To solve this, we **L2-normalize each feature group individually** first.
   - We then concatenate these normalized groups to form a unified, multi-faceted embedding vector (114 dimensions).
   - Finally, we **L2-normalize the entire concatenated embedding**. This ensures that each feature modality contributes equally and robustly to the final similarity metric.

3. **Cosine Similarity Computation**:
   - Because the final embeddings are L2-normalized (unit Euclidean norm), the cosine similarity between the query embedding and any library track embedding is exactly equivalent to their dot product.
   - This dot product is guaranteed to lie in $[-1.0, 1.0]$.

4. **Ranking & Output**:
   - Every library track in `/workspace/library/` is processed, and its cosine similarity with `/workspace/query.wav` is computed.
   - The results are sorted in descending order of similarity and written to `/workspace/results.json`.
